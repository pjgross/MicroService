Peter gross : NextJS Authorisation Approach

Created by Peter Gross, last modified on Sep 13, 2020



For a standard client side rendered application there are quite a few communication going on with the server, and we only really need to do the authorisation on the request to get data from the server

However in a Server Side rendered page the browser only makes a single request, so there is no chance to insert headers unless the link was requested by our app itself.

This means that if any headers such as authorisation tokens need to sent over with the first request

The mechanism for doing this is via cookies which we will use as the transport mechanism for the information and JWT tokens for encoding the authentication information.
