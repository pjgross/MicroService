This section highlights the design considerations I learnt while developing this example application


The basic design is to have a system where:

  • Any user can put tickets up for sale for a price
  • Anyone can create an order for a ticket, which would
    • lock other users out from buying the ticket
    • have a timer so the order would be cancelled after 15 minutes if not paid for
  • Allow the user to pay for the order
  • Provide Authentication that can cope with client and server side rendering
  • Have all of these services only talk through asynchronous event bus messaging
  • Have the clients only talk to a single end point, so a proxy service will be needed
  • Have each service handle its own REST api routing, application logic and database management
