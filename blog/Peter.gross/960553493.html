Peter gross : 6) Decentralised Data Management

Created by Peter Gross, last modified on Sep 15, 2020


Traditional systems use a single database that stores all of the data for each business process it implements


In Microservices, each service should have its own database if it to be decoupled from the other services


Important notes:

  • This is the most controversial attribute of Microservices
  • Not always possible
  • Raises problems such as distributed transactions, data duplication and more
  • If services share a database then there needs to be clear ownership defined


Motivation

  • Right tool for the right task
  • Enables isolation of the services
