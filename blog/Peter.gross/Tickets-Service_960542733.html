Database Design

For an Overview of the Typescript approach to MongoDB read thisÂ page

The main design issue we had to cater for, was the version number increments on each update to the ticket record

The first thing we had to do was import the mongoose module that will update the version number for us

import { updateIfCurrentPlugin } from "mongoose-update-if-current"

To make the version number field easier to read, we need to change the default field of __v to version

ticketSchema.set("versionKey", "version")

Then we enable the module we imported

ticketSchema.plugin(updateIfCurrentPlugin)

The rest of the database model is fairly standard following the TypeScript approach we defined
