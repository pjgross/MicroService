Peter gross : 1) Componentisation via services

Created by Peter Gross, last modified on Sep 17, 2020


There are an number of ways of dividing up the code, either by placing code into libraries, or by creating software that exposes a public api that provides a service to the caller

Introduction

  • Modular design is always a good idea
  • Software can be broken down in seperate components
  • These components can be achieved using
    • Libraries - called directly within the process
    • Services - called by a public API (REST, Graphql or GRPc)


In a Monolith application all of the functionality is part of the single application

The Microservice architecture is about splitting out functionlity into seperate application that provide public api's to provide their services

You can still use libraries to provide modularity of common code, for example in NodeJS you can package code into npm modules but the design of the architecture is not based around these modules

The aim is to create smaller components that can be developed and deployed faster
